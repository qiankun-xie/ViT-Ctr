# Plan 02-02: Dataset Validation and Google Drive Upload

## Objective

验证生成的数据集质量（分布均匀性、失败率、数据完整性），上传到Google Drive供Colab访问，并记录数据集元信息。

## Tasks

### Task 1: 实现数据集验证脚本

<read_first>
- .planning/phases/02-large-scale-dataset-generation/02-CONTEXT.md (决策D-03, D-10)
- .planning/phases/02-large-scale-dataset-generation/02-RESEARCH.md (验证指标)
- data/*.h5 (Plan 02-01生成的HDF5文件)
</read_first>

<action>
创建 `scripts/validate_dataset.py`，包含以下验证功能：

1. **`check_distribution(h5_files)`**
   - 读取所有HDF5文件的labels数据集
   - 提取log10_Ctr列，绘制直方图（50个bins）
   - 检查是否有明显空白区域（任意bin计数<总数/100）
   - 打印Ctr范围: min, max, median
   - 保存直方图到 `data/ctr_distribution.png`

2. **`check_balance(h5_files)`**
   - 统计4种RAFT类型的样本数
   - 计算偏差: `max_count / min_count`
   - 如果偏差>1.1（超过10%不平衡），打印警告
   - 打印样本数表格

3. **`check_integrity(h5_file)`**
   - 验证fingerprints shape: (n, 64, 64, 2)
   - 验证labels shape: (n, 3)
   - 检查NaN/Inf: `np.isnan(fp).any()` 和 `np.isinf(fp).any()`
   - 检查Đ范围: labels[:, 2] 应全部 >= 1.0
   - 返回通过/失败状态

4. **`main()`**
   - 扫描 `data/*.h5` 文件
   - 对每个文件调用 `check_integrity()`
   - 调用 `check_distribution()` 和 `check_balance()`
   - 打印验证报告: 总样本数、失败率、分布质量、平衡性
   - 如果所有检查通过，打印 "✓ Dataset validation PASSED"
</action>
