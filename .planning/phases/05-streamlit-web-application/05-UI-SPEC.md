# Phase 5 — UI Design Contract

> Visual and interaction contract for the Streamlit web application. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Streamlit native) |
| Preset | not applicable |
| Component library | Streamlit 1.51+ built-in widgets |
| Icon library | none (no icons needed; Streamlit emoji shortcodes for minor decoration) |
| Font | Streamlit default (Source Sans Pro) |

Notes: Streamlit controls its own theme engine. Custom styling is limited to `st.markdown(unsafe_allow_html=True)` for the three result cards (D-06). No `.streamlit/config.toml` theme override needed — default light theme is appropriate for a research tool.

---

## Spacing Scale

Streamlit manages layout spacing internally via `st.columns`, `st.tabs`, `st.container`, and `st.divider`. The following scale applies to custom HTML blocks only (result cards, citation footer):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Not used |
| sm | 8px | Inner padding between label and value inside result cards |
| md | 16px | Card internal padding (all sides) |
| lg | 24px | Gap between result cards (handled by st.columns gap) |
| xl | 32px | Vertical margin above/below the result card row |
| 2xl | 48px | Not used |
| 3xl | 64px | Not used |

Exceptions: Streamlit's own widget spacing (data_editor, tabs, buttons) is not overridable and uses Streamlit's internal 1rem (~16px) rhythm. Do not fight it.

---

## Typography

Custom typography applies only inside `unsafe_allow_html` blocks. All other text uses Streamlit defaults.

| Role | Size | Weight | Line Height | Where Used |
|------|------|--------|-------------|------------|
| Body | 16px | 400 (regular) | 1.5 | Streamlit default — all widget labels, descriptions, st.write |
| Card value (Ctr) | 28px | 700 (bold) | 1.2 | Ctr result card primary number (D-06: visually prominent) |
| Card value (secondary) | 24px | 600 (semibold) | 1.2 | Inhibition period and retardation factor card numbers |
| Card label | 14px | 400 (regular) | 1.5 | Parameter name, CI text, log10 subtitle inside cards |

Font stack for custom HTML: `"Source Sans Pro", -apple-system, BlinkMacSystemFont, sans-serif` (matches Streamlit default).

---

## Color

Streamlit default light theme provides the dominant and secondary surfaces. Custom colors apply to result cards and error states only.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#ffffff` | Page background (Streamlit default) |
| Secondary (30%) | `#f0f2f6` | Streamlit widget backgrounds, data_editor, tab bar |
| Accent (10%) | `#ff4b4b` | Ctr card border only (2px solid); Predict button (Streamlit primary button uses this by default) |
| Destructive | not applicable | No destructive actions in this phase |

Accent reserved for:
- Ctr result card border (D-06: Ctr visually prominent)
- Streamlit primary button ("Predict") — uses Streamlit's built-in `type="primary"` which renders as `#ff4b4b`

Secondary cards (inhibition, retardation) use `#e0e0e0` 1px border — deliberately muted to create visual hierarchy per D-06.

### Heatmap Colormaps (D-09)
| Channel | Colormap | Rationale |
|---------|----------|-----------|
| Ch0 (Mn) | `viridis` | Perceptually uniform, colorblind-safe, standard for scientific data |
| Ch1 (Đ) | `plasma` | Perceptually uniform, visually distinct from viridis, colorblind-safe |

## Data Editor Column Config (Claude's Discretion)

| Column Key | Display Label | Type | min_value | max_value | format | help text |
|------------|--------------|------|-----------|-----------|--------|-----------|
| `[CTA]/[M]` | `[CTA]/[M]` | NumberColumn | 0.0 | — | `%.4f` | `Molar ratio of CTA to monomer (e.g., 0.005)` |
| `conversion` | `Conversion` | NumberColumn | 0.0 | 1.0 | `%.3f` | `Monomer conversion (0 to 1)` |
| `Mn` | `Mn (g/mol)` | NumberColumn | 0.0 | — | `%.0f` | `Number-average molecular weight in g/mol` |
| `D` | `Đ (Mw/Mn)` | NumberColumn | 1.0 | — | `%.3f` | `Dispersity (Mw/Mn), must be >= 1.0` |

Internal column names use ASCII (`D` not `Đ`) to avoid encoding issues (RESEARCH.md Pitfall 6). Display labels use Unicode via `column_config`.

## Interaction States

| State | Trigger | Visual |
|-------|---------|--------|
| Initial load | Page open | Title, subtitle, monomer selector, empty data_editor in Manual tab, Predict button disabled-looking (no data) |
| Data entered | User fills >= 1 row | Data visible in editor; Predict button clickable |
| Validation failure | Click Predict with invalid data | `st.error()` red banner listing all errors; no results shown |
| Prediction running | Click Predict with valid data | `st.spinner("Running prediction with 200 bootstrap samples...")` |
| Results displayed | Prediction complete | Three cards + ctFP heatmap appear below button |
| File uploaded | Upload Excel/CSV in File Upload tab | Data populates into data_editor; user can review/edit before Predict |
| Missing checkpoints | App startup, files absent | `st.error()` banner at top; Predict button hidden or disabled |
| No bootstrap | bootstrap_heads.pth absent | `st.warning()` banner; results show point estimates only, no CI |
