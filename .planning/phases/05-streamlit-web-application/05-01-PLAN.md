    **src/app_utils.py must contain these 4 functions:**

    1. `validate_input(df: pd.DataFrame) -> tuple[pd.DataFrame | None, list[str]]`
       - Drop rows where ALL columns are NaN (empty rows from data_editor)
       - Drop rows where ANY required column is NaN (incomplete rows)
       - Check len(df_valid) >= 3, error: "At least 3 complete data points required (found {n})."
       - Per-row validation (use 1-based row numbering from df_valid.index + 1):
         - conversion not in (0, 1): "Row {n}: conversion must be in (0, 1), got {value:.3f}."
         - Mn <= 0: "Row {n}: Mn must be > 0, got {value:.1f}."
         - D < 1.0: "Row {n}: Dispersity (Đ) must be >= 1.0, got {value:.3f}."
         - [CTA]/[M] <= 0: "Row {n}: [CTA]/[M] must be > 0, got {value:.6f}."
       - Return (df_valid, errors). If errors non-empty, still return df_valid (caller decides whether to proceed).
       - Exception: if < 3 points, return (None, errors) — fatal.
       - Error message format MUST match 05-UI-SPEC.md Copywriting Contract exactly.

    2. `prepare_ctfp_input(df: pd.DataFrame, m_monomer: float) -> list[tuple]`
       - df columns: '[CTA]/[M]', 'conversion', 'Mn', 'D'
       - cta_ratio_norm = df['[CTA]/[M]'] / 0.1 (normalize to [0,1] range per RESEARCH.md Pattern 1)
       - mn_theory = m_monomer / df['[CTA]/[M]'] (per-row, since each row may have different [CTA]/[M])
       - mn_norm = df['Mn'] / mn_theory
       - Return list of (cta_ratio_norm, conversion, mn_norm, dispersity) tuples
       - This output feeds directly into ctfp_encoder.transform()

    3. `generate_template() -> BytesIO`
       - Create DataFrame with columns ['[CTA]/[M]', 'conversion', 'Mn', 'D'] (ASCII column names per UI-SPEC)
       - 3 example rows: (0.005, 0.10, 15000, 1.15), (0.005, 0.30, 42000, 1.20), (0.005, 0.50, 68000, 1.25)
       - Write to BytesIO via df.to_excel(buf, index=False, engine='openpyxl')
       - Return buf (seeked to 0)

    4. `format_results(mean_pred: np.ndarray, half_width: np.ndarray | None) -> dict`
       - mean_pred shape (3,): [log10_Ctr, inhibition_period, retardation_factor]
       - half_width shape (3,) or None (point estimate mode)
       - Ctr back-transform: ctr = 10 ** log10_ctr (per D-07)
       - Ctr CI is ASYMMETRIC in original scale: ctr_lower = 10^(log10_ctr - hw[0]), ctr_upper = 10^(log10_ctr + hw[0])
       - Inhibition and retardation CI are symmetric: value +/- hw
       - If half_width is None: all CI fields are None
       - Return dict with keys: 'ctr', 'log10_ctr', 'ctr_lower', 'ctr_upper', 'inhibition_period', 'inh_lower', 'inh_upper', 'retardation_factor', 'ret_lower', 'ret_upper'

    **IMPORTANT anti-patterns to avoid:**
    - Do NOT compute Ctr CI as `ctr +/- 10^hw` — must transform bounds through 10^x individually (RESEARCH.md Pattern 5)
    - Do NOT use raw Mn in prepare_ctfp_input output — must divide by Mn_theory (RESEARCH.md Pitfall 1)
    - Do NOT use Unicode Đ in DataFrame column names — use ASCII 'D' internally (RESEARCH.md Pitfall 6)
